# 14. Observability

## Goals

Operators must see:

- write acknowledgement latency;
- cache hit/miss behavior;
- target-N repair lag;
- latency graph health;
- spot churn impact;
- global replication lag;
- conflict rates;
- graph/vector query cost;
- vector index freshness.

## Structured logs

Every log line should include:

```json
{
  "cluster_id": "prod-use1",
  "member_id": "m1",
  "storage_uuid": "s1",
  "component": "replication",
  "event": "origin_plus_one_ack_failed",
  "namespace": "default",
  "key_hash": "...",
  "request_id": "..."
}
```

Do not log raw keys or values by default.

## Admin endpoints

These HTTP `/admin/*` endpoints are a **thin read-only/debug gateway over `AdminService`** (the gRPC
management surface defined in `11_api_contracts.md`), for humans and dashboards. They are not a second
mutation path: state-changing operations (`TriggerRepair`, drain, lifecycle) go through `AdminService`
gRPC and require the operator/admin role (`15_security.md`). Keep the HTTP surface read-only.

### Membership

```text
GET /admin/members
```

Returns liveness, node, zone, region, geo, storage UUID.

### Latency graph

```text
GET /admin/latency-graph
```

Returns top edges, stale edges, and placement scores.

### Key debug

```text
GET /admin/key/{namespace}/{base64_key}
```

Returns holder list, versions, conflict state, TTL, subscribers.

Require admin auth. Redact value by default.

### Repair status

```text
GET /admin/repair
```

Returns under-replication estimates and queue state.
